## 1 如何界定开源项目还是内部项目

----

| 开源项目       | 内部项目（或内部开源）  |
| ------------- | -------------     |
| 与公司业务无关 | 与公司业务有关 |
| 原子化，容易复用 | 通用项目，方便接入 |
| 注意敏感信息 | 配置相关变量 |
| 代码抽象要求高 | 通用性和兼容性要求高 |
| 单测要求高 | 单测视具体情况定，集成测试要求高 |

----
